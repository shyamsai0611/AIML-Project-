Variable Naming:
NEVER LEAVE VAR NAMES WITH VAGUE NAMES: {CAN CHANGE LATER}
-> Global variables - ALL_CAPS
-> Class / Compundnames - PascalCaseNaming
-> Function name - camelCaseNaming
-> Local variables, Parameters  - all_smalls_with_underscore
-> Leftovers - Capital_with_underscore
-> Ex: if var name is count then make it count_
    local and param - countL / countP


Coding:
-> Strictly no input in function or modules. If needed then either: parameters or setters and getters
-> Break things into different files
-> Make code generic
